# 초음파 센서 기반 자율주행 직각 주차 가이드라인

본 문서는 성균관대학교 Automation Lab에서 제공한 초음파 센서 기반 자율주행 직각 주차 알고리즘의 설계 가이드를 상세하게 정리한 보고서입니다. 카메라 없이 오직 초음파 센서만을 활용하여 주차공간 내 장애물 차량들 사이에 직각 주차를 안전하게 완료하는 것을 목표로 합니다.

---

## 1. 공지 사항 및 개요 (Notice & Overview)

- **자료의 성격**: 본 가이드라인은 학생들의 알고리즘 이해를 돕기 위한 예시 자료입니다. 대회 규정 등의 실제 미션 수행 시에는 반드시 **공식 배포된 규정집을 우선**으로 해야 합니다.

- **센서 수량 차이**:

    - 본 가이드 영상의 테스트 차량에는 총 12개의 초음파 센서가 사용되었으나, 실제 자율주행 AI 칩 설계 참가자들에게 제공되는 초음파 센서는 **총 5개**입니다.

    - 따라서 참가자들은 본 가이드라인의 알고리즘 흐름을 바탕으로, 5개의 초음파 센서를 차량의 어느 위치에 배치하고 어떻게 조합하여 사용할지 최적의 하드웨어 및 소프트웨어 설계를 직접 고민해야 합니다.

> **[구현 확정] 5개 센서 선정 결과**
>
> 가이드의 12개 센서 중 **0, 2, 8, 9, 11번** 위치를 선택했습니다. 선정 근거는 다음과 같습니다.
>
> - 원본 알고리즘이 요구하는 역할은 ① 우측면 공간 탐색·평행 정렬, ② 좌측 대칭 비교(중앙 정렬), ③ 후방 마무리 판단의 세 가지입니다.
> - 우측면은 **전·후 2개(2, 0번)** 가 반드시 필요합니다. 하나로는 평행 여부(각도)를 알 수 없기 때문입니다.
> - 중앙 정렬에는 좌측 대칭점인 **8번**이 필요합니다.
> - 후진 마무리에는 후방 코너 **9, 11번**이 필요합니다.
> - 결과적으로 5개가 정확히 소진되며, **전방 감시 센서는 두지 않습니다.** 원본 가이드의 3, 4, 5번(전방 코너·전면 중앙)이 담당하던 역할은 포기합니다. 본 미션의 주행 경로상 전방 장애물이 없다는 전제이며, 이는 의식적으로 감수한 설계 선택입니다.

## 3. 초음파 센서 배치도 및 번호 구성

가이드 영상의 예시 차량에는 반시계 방향으로 **0번부터 11번까지 총 12개**의 초음파 센서가 부착되어 있습니다. 각 센서의 위치와 방향은 다음과 같습니다.

|센서 번호|부착 위치 및 측정 방향|역할 (알고리즘 상 쓰임새)|
|---|---|---|
|**0**|우측 후측면 (Right Rear-Side)|주차 공간 탐색, 전진 정지 시점 판단, 평행 맞추기 및 중앙 정렬|
|**1**|우측 중앙 (Right Mid-Side)|주차 공간 탐색 및 진입 판단|
|**2**|우측 전측면 (Right Front-Side)|주차 공간 탐색, 후진 진입 시 정지 판단, 평행 맞추기|
|**3**|우측 전면 코너 (Right Front-Corner)|전방 코너 장애물 감지|
|**4**|전면 중앙 (Front Center)|전방 장애물 감지|
|**5**|좌측 전면 코너 (Left Front-Corner)|전방 코너 장애물 감지|
|**6**|좌측 전측면 (Left Front-Side)|좌측 전방 장애물 감지|
|**7**|좌측 중앙 (Left Mid-Side)|좌측 측면 거리 유지|
|**8**|좌측 후측면 (Left Rear-Side)|중앙 정렬 (우측 0번 센서와 비교)|
|**9**|좌측 후면 코너 (Left Rear-Corner)|후진 마무리 정지 판단 (후방 장애물 감지)|
|**10**|후면 중앙 (Rear Center)|후방 장애물 감지|
|**11**|우측 후면 코너 (Right Rear-Corner)|후진 마무리 정지 판단 (후방 장애물 감지)|

_※ 차량 전방은 이미지상 우측 방향(4, 5번이 있는 곳)이며, 이에 맞춰 반시계 방향으로 센서가 배치되어 있습니다._

### 3-1. [구현 확정] 실제 사용 센서 5개

| 이름 | 위치 | 부착 방향 | 사용되는 단계 |
|---|---|---|---|
| **S0** | 우측 후측면 | 차체 우측 직각 | 2, 3, 5단계 + 출차 |
| **S2** | 우측 전측면 | 차체 우측 직각 | 2, 3, 4, 5단계 |
| **S8** | 좌측 후측면 | 차체 좌측 직각 | 출차 |
| **S9** | 좌측 후방 코너 | **후방 바깥쪽 45도** | 7단계 + 후진 충돌 방지 |
| **S11** | 우측 후방 코너 | **후방 바깥쪽 45도** | 7단계 + 후진 충돌 방지 |

**S9·S11의 부착 각도가 알고리즘 성립의 전제입니다.** 이 둘을 차량 정후방을 향하도록 붙이면 주차 공간 뒤편에 벽이 없는 한 아무것도 감지하지 못해, 7단계 정지 시점을 영영 판단할 수 없습니다. **후방 바깥쪽 45도**로 붙여야 양옆 장애물 차량의 측면을 감시하다가, 차량이 충분히 깊이 들어가 그 차들의 뒷단을 지나치는 순간 감지가 풀리며 이것이 정지 신호가 됩니다.

### 3-2. [구현 확정] 부착 시 필수 실측 항목

**S0와 S2 사이의 차체상 간격**이 두 장애물 차량 사이의 실제 빈 간격보다 **반드시 작아야 합니다.**

2단계 공간 탐색은 "S2와 S0가 **동시에** 빈 공간을 가리키는 순간"을 포착하는 방식입니다. 만약 S0–S2 간격이 두 차 사이 간격보다 크면 두 센서가 동시에 비는 순간이 물리적으로 존재하지 않아, 차량은 주차 공간을 그대로 지나칩니다.

| 실측 항목 | 확인 방법 |
|---|---|
| S0–S2 간격 | 차체에 부착한 두 센서 사이 거리를 자로 실측 |
| 두 장애물 차량 사이 실제 빈 간격 | 연습 트랙에서 실측 (주차 공간 950mm + 양옆 여유) |

간격 확보가 불가능한 경우의 대안은 "동시에 빈 순간" 대신 "S2가 빈 뒤 일정 시간 경과"로 바꾸는 것이지만, 시간 의존이라 출발 위치 추첨에 취약하므로 권장하지 않습니다. **센서를 최대한 안쪽으로 좁게 붙이는 것이 정답입니다.**

또한 규정상 **모든 센서는 차량의 전후 110cm, 좌우 60cm, 높이 75cm를 넘어 부착할 수 없으므로**, 지그 제작 시 이 범위를 반드시 지켜야 합니다.

## 4. 단계별 직각 주차 알고리즘 상세 (Detailed Step-by-Step)

카메라 없이 초음파 센서만으로 주차하기 때문에, 차량은 거리 값의 변화를 감지하여 자신이 주차 과정의 어느 단계(State)에 와 있는지를 지속적으로 파악해야 합니다. 본 알고리즘은 총 7개의 단계로 구성됩니다.

```
[1단계: 직진 주행] ──> [2단계: 공간 감지 후 정지] ──> [3단계: 좌측 조향 후 전진]
                                                               │
[6단계: 중앙 정렬] <── [5단계: 평행 정렬 및 후진] <── [4단계: 우측 조향 후 후진]
     │
[7단계: 후방 주차 완료]
```

### 1단계 : 핸들 중립 후 직선 주행

- **동작**: 시작 위치에서 핸들(조향)을 중립으로 설정하고 앞으로 곧게 직선 주행합니다.

- **목적**: 주차 공간이 나타날 때까지 안정적으로 전진합니다.

> **[구현 확정]**
> 1단계와 2단계는 하나의 상태 `SEARCH_GAP`으로 통합합니다. 별도의 상태로 나눌 필요가 없으며, "중립 조향으로 전진하면서 공간을 탐색한다"가 하나의 동작이기 때문입니다.
>
> - 조향 목표: `0` (중립). 가변저항 폐루프로 도달시킨 뒤 전진 시작
> - 구동: `SPEED_FORWARD` 듀티로 좌우 후륜 **동일 속도** 전진
> - 전방 센서가 없으므로 이 구간은 전방 감시 없이 진행됩니다

### 2단계 : 두 차 사이에서 정지 (센서 사용: 0, 1, 2)

- **동작**: 직선 주행을 하다가 주차되어 있는 두 장애물 차량 사이(빈 주차 공간)를 지나칠 때 정지합니다.

- **판단 조건**: 우측 측면에 있는 **0번, 1번, 2번 센서값**이 모두 설정해 둔 임계치(Threshold) 이상으로 증가할 때 정지합니다.

    - _이유_: 센서 거리 값이 커진다는 것은 우측에 주차 공간(빈 공간)이 나타나 장애물과의 거리가 멀어졌음을 의미합니다.

> **[구현 확정] 1번 센서 부재에 따른 재설계**
>
> 1번(우측 중앙)이 없으므로 **S2와 S0 두 개만으로** 판단합니다. 단순히 "값이 임계치 이상"이 아니라 **거리 변화의 시간적 순서**를 봅니다. 절대 거리값은 신뢰도가 낮지만 "멀다 → 가깝다 → 다시 멀다"는 순서는 훨씬 견고하기 때문입니다.
>
>차량이 전진하며 겪는 신호 순서:

| 시점  | S2 (전측면) | S0 (후측면) | 의미                      |
| --- | -------- | -------- | ----------------------- |
| ①   | 아주 멀     | 아주 멀     | 빈 도로 주행 중               |
| ②   | 감지       | 아주 멀     | 첫 장애물 차량의 앞부분에 진입       |
| ③   | 감지       | 감지       | 첫 장애물 차량 옆을 통과 중        |
| ④   | 아주 멀     | 감지       | 차량 앞부분이 주차 공간에 진입       |
| s⑤  | 아주 멀     | 아주 멀     | 차체 전체가 주차 공간 앞에 정렬 → 정지 |

> 판정 로직 (2단계 래치):
>
> 1. `S2`가 한 번이라도 **감지**되면 "첫 장애물 차량을 봤다" 플래그를 세운다
> 2. 플래그가 선 이후, `S2`와 `S0`가 **동시에 아주 멈**이 되면 정지
>
> 플래그를 두는 이유는, 출발 직후 아직 아무것도 못 본 상태에서도 두 센서가 모두 비어 있어 조건이 즉시 만족되기 때문입니다.

### 3단계 : 좌측 조향 후 후진 시작 위치까지 직진 (센서 사용: 0)

- **동작**: 정지한 위치에서 조향을 최대 좌측(최좌측)으로 꺾은 상태로 앞으로 주행(직진)하다가 정지합니다.

- **판단 조건**: 우측 후측면의 **0번 센서값**이 일정 값 이상이 될 때 정지합니다.

    - _이유_: 차량이 앞·좌측 방향으로 빠져나가면서 차량 뒷부분(0번 센서 부근)이 기존 장애물 차량에서 완전히 멀어지는 시점을 포착하여 후진 기어를 넣을 최적의 시작 위치를 잡는 것입니다.

> **[구현 확정]** 상태명 `SWING_LEFT`
>
> - 조향 목표: `-STEER_FULL` (최대 좌조향). **바퀴가 목표 각도에 도달한 뒤 전진을 시작**합니다 (블로킹 조향)
> - 구동: `SPEED_FORWARD`로 전진
> - 종료 판정: **`S0`가 다시 감지되었다가 → 다시 아주 멈**이 될 때 정지
>
> 원본 판단 기준인 **0번(S0, 우측 후측면)** 을 그대로 따릅니다. 2단계와 동일한 "감지 → 해제" 2단계 래치를 사용합니다.
>
> **⚠️ 주의**: `SEARCH_GAP`은 "S2·S0가 **모두** 아주 멈"일 때 정지하므로, `SWING_LEFT`에 진입하는 순간 S0는 이미 '해제' 상태입니다. 따라서 이 래치의 '감지' 단계는 좌조향 스윙 중에 **S0가 두 번째 장애물 차량을 다시 감지**하는 것에 의존합니다. 좌조향으로 차가 좌측으로 빠지는 동안 후측면 S0가 두 번째 차를 재감지하지 못하면 종료되지 않고 타임아웃으로 빠지므로, **트랙에서 이 재감지가 실제로 일어나는지 반드시 확인**해야 합니다. (재감지가 안 되면 전측면 S2 기준으로 되돌리는 것을 검토하십시오.)

### 4단계 : 우측 조향 후 두 차 사이에 들어올 때까지 후진 (센서 사용: 2)

- **동작**: 3단계 정지 위치에서 조향을 최대 우측(최우측)으로 꺾은 상태로 후진하다가 주차 공간 입구 근처에 들어섰을 때 정지합니다.

- **판단 조건**: 우측 전측면의 **2번 센서값**이 일정 값 이하가 될 때 정지합니다.

    - _이유_: 차가 회전하며 주차 영역 내부로 엉덩이를 밀어 넣을 때, 우측 앞 범퍼 측면(2번 센서)이 주차 공간 우측에 있는 장애물 차량과 가까워지는 정렬 타이밍을 잡기 위함입니다.

> **[구현 확정]** 상태명 `REVERSE_RIGHT`
>
> - 조향 목표: `+STEER_FULL` (최대 우조향). 바퀴 도달 후 후진 시작
> - 구동: `SPEED_REVERSE`로 후진
> - 종료 판정: **`S2`가 (한 번 해제된 뒤) 다시 감지**되면 정지
>
> 원본의 **2번(S2, 우측 전측면)** 단독 판정을 따릅니다. 우조향 후진으로 엉덩이를 밀어 넣을 때 우측 앞 범퍼 측면(S2)이 주차 공간 우측 장애물 차량과 가까워지는 시점을 잡습니다.
>
> 다만 순수 단일조건("S2가 감지되면 정지")은 진입 직후 S2 잔상/근접으로 **즉시 오종료**될 수 있습니다. median 창이 직전 구간 값을 들고 있고, 진입 시점에 S2가 이미 우측 차에 가까울 수 있기 때문입니다. 이를 막기 위해 **"먼저 S2가 해제된 것을 확인한 뒤 다시 감지되면 정지"** 하는 해제→감지 2단계 가드를 둡니다.
>
> **후진 중에는 S9·S11을 상시 충돌 감시**에 사용합니다 (7-4절 참조).

### 5단계 : 두 차와 평행하도록 조향 좌우로 보정하며 후진 (센서 사용: 0, 2)

- **동작**: 주차 공간에 들어가면서 차가 비스듬하거나 한쪽으로 치우치지 않도록 조향을 좌우로 미세하게 움직이며 정밀하게 후진합니다.

- **판단 조건**: **0번 센서(우측 후방 측면)와 2번 센서(우측 전방 측면) 값을 실시간 비교**합니다.

    - _이유_: 차량이 장애물 차량과 완전히 평행해지면, 측면의 두 센서(0번과 2번) 거리 값이 거의 같아집니다. 두 값의 차이가 최소화되어 평행 상태가 확인되면 다음 단계로 넘어갑니다.

### 6단계 : 두 차 중앙에 위치하도록 거리 조정 (센서 사용: 0, 8)

- **동작**: 차량이 좌우 장애물 차량의 한가운데 위치하여 문콕을 방지하고 안정적인 대칭을 이루도록 측면 공간을 조정합니다.

- **판단 조건**: 우측의 **0번 센서**와 좌측의 **8번 센서** 값을 실시간 비교하여 좌우 여유 공간을 맞춥니다.

    - _이유_: 0번 센서값과 8번 센서값이 거의 대칭(동일)을 이룬다면 두 장애물 차량의 정중앙에 올바르게 위치한 것으로 간주합니다.

> **[구현 확정] 5단계(+7단계 종료) — 상태명 `ALIGN_REVERSE`. 6단계는 제거.**
>
> **원본 5단계(S0·S2 각도 정렬)만 수행하고, 6단계(S0·S8 중앙 정렬)는 제거합니다.** 자동차는 조향이라는 입력 하나로 움직이는데(비홀로노믹) 각도(평행)와 좌우 위치(중앙) 두 목표를 동시에 맞출 수 없습니다. 이번 설계의 목표는 **정렬 과정을 최대한 단순하게** 만드는 것이므로, 좌우 위치(중앙 정렬)를 포기하고 **각도만** 맞춥니다.
>
> ```
> err = S0 − S2                          우측 두 센서의 거리차 = 차체 기울기
> |err| ≤ ALIGN_DEADBAND_CM  →  조향 0 (중립)
> err > 0                     →  조향 +STEER_ALIGN_STEP
> err < 0                     →  조향 −STEER_ALIGN_STEP
> ```
>
> 차량이 우측 장애물 차량과 평행해지면 S0(후측면)와 S2(전측면)의 거리값이 거의 같아집니다. 두 값의 차이(각도 오차)를 줄이는 방향으로 조향을 미세 조정하며 저속 후진합니다.
>
> **⚠️ 중앙 정렬을 없앴으므로 주차가 좌우 한쪽으로 치우칠 수 있습니다.** 이는 의식적으로 감수한 단순화입니다. 나중에 중앙 정렬이 필요해지면 캐스케이드가 아닌 별도 방식으로 다시 설계합니다.
>
> **조향 단수**: 이 단계에서는 최대 조향(±7)이 아니라 **±2 수준의 작은 조향**을 사용합니다. 다만 주차 공간 깊이(1500mm)가 제한되어 있어 ±2로는 각도가 다 펴지기 전에 공간이 끝날 수 있으므로, `STEER_ALIGN_STEP`을 상수로 분리해 **2 ~ 4 사이에서 실측 튜닝**합니다.
>
> **`ALIGN_SIGN` 주의**: 후진 중에는 조향 방향이 만드는 차체 회전 방향이 전진과 반대입니다. 부호를 책상에서 결정할 수 없으므로 `ALIGN_SIGN`(+1/−1) 상수로 빼두고, **실주행에서 오차가 줄지 않고 커지면 뒤집습니다.**
>
> **밴드를 쓰지 않습니다**: 7-2절의 5단계 밴드는 이벤트 판정 전용이며, 이 정렬 제어에는 **필터링된 연속값(cm)** 을 그대로 씁니다. 두 센서가 똑같이 "가까움"으로 뭉개지면 차량이 몇 도 틀어졌는지 정보가 통째로 사라지기 때문입니다.
>
> **후진 종료 판정은 7단계 내용을 그대로 사용합니다** (아래 7단계 블록 참조). 정렬 후진 중에는 45도 후방 코너 센서 S9·S11이 양옆 차량 측면을 감지하고 있다가, 차량이 충분히 깊이 들어가 두 센서 모두 감지가 풀리는 것을 종료 신호로 삼습니다.

### 7단계 : 후진하여 주차 마무리 (센서 사용: 9, 11)

- **동작**: 중앙 정렬이 끝난 후, 핸들을 곧게 펴고 주차 칸 안쪽 깊숙이 똑바로 후진합니다.

- **판단 조건**: 후방 코너의 **9번 및 11번 센서**를 모니터링하다가, 이 센서값들이 특정 임계값 이상(또는 벽에 안전하게 근접한 특정 신호 범위)이 도달하면 차량을 멈추고 주차를 최종 완료합니다.

> **[구현 확정] 7단계는 `ALIGN_REVERSE`의 종료 조건으로 흡수**
>
> 별도 상태를 두지 않고, 정렬 후진을 계속하다가 아래 조건이 만족되면 정지합니다.
>
> **종료 판정 (2단계 래치)**:
> 1. 진입 후 `ALIGN_MIN_SECONDS` 경과 + `S9` 또는 `S11`이 **감지**됨 → 플래그 세움
> 2. 플래그가 선 이후, `S9`와 `S11`이 **모두 아주 멈**이 되면 정지
>
> 45도로 붙은 후방 코너 센서가 양옆 차량의 측면을 보다가, 차량이 충분히 깊이 들어가 그 차들의 뒷단을 지나치면 동시에 감지가 풀립니다. 이것이 "양옆 차량과 깊이가 맞춰졌다"는 신호입니다.
>
> **⚠️ 2단계 래치가 없으면 주차가 실패합니다.** 정렬 후진에 막 진입한 시점에는 S9·S11이 아직 옆 차량을 보지 못해 "비어 있음"으로 읽힙니다. 곧바로 종료 판정을 하면 **깊이를 전혀 확보하지 못한 채 주차가 끝나버립니다.** 게다가 중앙값 필터가 직전 단계의 값을 창에 들고 있어 이 오판 위험이 더 커집니다. 반드시 "감지 → 해제" 순서를 확인해야 합니다.

## 5. 초음파 센서 사용 시 주의사항 및 물리적 특성

초음파 센서는 가격이 저렴하고 거리 측정이 쉽지만, 음파의 반사 특성 때문에 하드웨어적 한계가 존재하므로 알고리즘 설계 시 이를 반드시 감안해야 합니다.

```
[굴곡진 부분 측정]                     [평평한 부분 측정]
   (초음파 반사 분산)                     (안정적인 수직 반사)
      장애물 차량                            장애물 차량
       /=======\                              |=======|
      /         \                             |       |
     *    <-  -  - (오차 발생)                 * <- - - (정확함)
    /             \                          |       |
   차량            차량                       차량    차량
```

1. **굴곡진 부분에서의 거리 인식 불량 (난반사)**

    - 장애물 차량의 모퉁이나 범퍼 하단의 둥근 곡면(굴곡진 부분)에 초음파가 닿으면, 음파가 센서로 똑바로 되돌아오지 않고 다른 방향으로 튀어나가는 **난반사 현상**이 발생합니다.

    - 이로 인해 실제 거리보다 아주 먼 거리로 측정되거나 거리 감지가 되지 않는 오류가 생길 수 있습니다.

2. **평평한 부분에서의 높은 정확도 (정반사)**

    - 장애물 차량의 측면 문짝이나 뒤쪽 범퍼의 평평한 수직 면에 쏘아진 초음파는 정반사되어 높은 신뢰도의 거리 측정 결과를 보장합니다.

3. **알고리즘 설계 시 팁**

    - 곡면 부근을 지나갈 때는 일시적으로 발생하는 튀는 센서값(노이즈)을 무시하는 필터링(예: 칼만 필터, 이동 평균 필터 등)을 적용해야 합니다.

    - 주차 진입 및 정지 시점 계산 시, 최대한 평평한 벽면에 대응하는 센서들의 데이터를 적극적으로 신뢰하여 임계치를 설정하는 것이 유리합니다.

> **[구현 확정] 필터 선택 — 이동평균이 아니라 중앙값(median)**
>
> 원본은 칼만 필터나 이동평균을 예로 들었으나, **본 미션에서는 이동평균(및 EMA) 계열을 사용하지 않습니다.**
>
> 주차 과정에서 장애물 옆을 지나가며 거리가 급변하는 것은 **노이즈가 아니라 진짜 신호**입니다. 30cm에서 감지 없음으로 뚝 끊기는 것이 정상인데, 이동평균·EMA는 경계를 뭉개는 것이 본질이라 이미 지나간 장애물을 계속 붙들고 있게 만듭니다.
>
> **중앙값 필터(median)** 는 튀는 값은 걸러내면서 계단 변화는 그대로 통과시키는 edge-preserving 필터라 이 용도에 정확히 맞습니다.
>
> **창 크기는 5**를 사용합니다. 3은 고립된 이상치 1개만 잡아내는데, 원문에도 나오듯 밀폐 공간에서는 의도치 않은 반사가 늘고 센서 개수가 많을수록 오류도 함께 증가합니다. 주차장은 벽에 둘러싸인 밀폐 공간이고 센서도 5개라 **연속 2회 오류가 충분히 발생 가능**하므로 창 5개가 필요합니다.
>
> **"변화량 제한 필터"는 쓰지 않습니다.** 유령 에코를 잡으려면 임계값이 작아야 하고 장애물 경계의 진짜 급변을 허용하려면 커야 하는데, 둘 다 "한 프레임에 크게 변하는 값"이라 구분이 불가능합니다. 중앙값 필터가 이 역할을 부작용 없이 대신합니다.

## 6. Parking Logic State Machine

단계별 주차 프로세스를 소프트웨어로 안정적으로 구현하기 위해 **State Machine (상태 머신)** 구조를 활용합니다. `switch-case` 문을 사용하여 가독성과 디버깅 효율을 높인 설계 뼈대입니다.

### State Flow Diagram

```
[NEUTRAL] ---> [FORWARD] ---> [STRAIGHT_LEFT] ---> [REVERSE_RIGHT]
                                                          |
                                                          v
   [END]   <--- [AREA_CORRECTION] <--- [HORIZONTAL_CORRECTION]
```

### 코드 구조 예시 (C/C++)

```
switch (currentState) {
    case NEUTRAL:
        // Step 1: 핸들이 중립 상태인지 확인
        if (핸들이 중립 상태) {
            currentState = FORWARD;
        }
        break;

    case FORWARD:
        // Step 2: 전진 중 주차 공간 감지
        전진();
        if (두 차 사이에 위치 && 빈 공간 감지) {
            // S0, S1, S2 센서값 기반 판단
            정지();
            currentState = STRAIGHT_LEFT;
        }
        break;

    case STRAIGHT_LEFT:
        // Step 3: 좌조향 후진 시작 위치까지 전진
        좌회전 상태로 전진();
        if (후진할 위치에 도달) {
            // S0 센서값 기반 판단
            정지();
            currentState = REVERSE_RIGHT;
        }
        break;

    case REVERSE_RIGHT:
        // Step 4: 우조향 후 주차 공간 진입 후진
        우회전 상태로 후진();
        if (두 차 사이에 도달) {
            // S2 센서값 기반 판단
            정지();
            currentState = HORIZONTAL_CORRECTION;
        }
        break;

    case HORIZONTAL_CORRECTION:
        // Step 5: 평행 맞추기 보정
        좌우로 움직이며 후진();
        if (두 차와 평행한 상태) {
            // |S0 - S2| 편차 기반 판단
            정지();
            currentState = AREA_CORRECTION;
        }
        break;

    case AREA_CORRECTION:
        // Step 6 & 7: 정중앙 정렬 및 후진 마무리
        우회전 또는 좌회전 상태로 전진/후진하며 미세 조정();
        if (두 차의 중앙에 위치) {
            // S0 == S8 기반 판단 후 최종 후진 완료
            currentState = END;
        }
        break;

    case END:
        // 주차 종료 상태
        직각 주차 완료 후 모든 모터 정지();
        break;
}
```

## 6. 결론 및 참가자 안내

- 본 가이드는 풍부한 센서(12개)를 바탕으로 각 상황별로 역할을 나누어 주차 로직을 설계했습니다.

- 참가자 여러분은 **5개의 센서만 제공**받으므로, 아래와 같은 효율화 과정을 거쳐야 합니다.

    - **대칭의 최소화**: 예컨대 우측 센서 중심으로 주차 공간을 탐색한다면 주차 센서의 우선순위를 우측면에 더 많이 부여할 수 있습니다.

    - **역할 통합**: 전/측면 정지 조건과 평행 조건을 하나의 센서가 각 State 마다 다르게 해석하도록 소프트웨어적으로 구현할 수 있습니다. (예: State 2에서는 탐색용 센서로 작동, State 5에서는 평행 체크용 센서로 다중 활용)

- 위 알고리즘 단계를 토대로 `State Machine`을 구축하면 복잡한 자율주행 주차 미션을 안정적으로 구현할 수 있을 것입니다.

## 7. 규정에서 파생된 필수 요구사항

원본 가이드라인은 주차 완료(`END`)까지만 다루지만, 규정집상 미션 성공 판정은 그 이후까지 포함합니다.

### 7-1. 출차가 미션의 절반

> IN 지점 출발 → 후진으로 두 차량 사이에 수직 주차 → **3~5초 정차** → 빠져나와 **OUT 라인을 앞바퀴 중 한 바퀴라도 통과** → 미션 성공. 제한시간 4분.

원본 상태머신은 `END`에서 종료되므로, 아래 상태들을 직접 설계해 이어붙여야 합니다.

| 추가 상태 | 역할 |
|---|---|
| `HOLD` | 3~5초 정차 (규정 요구) |
| `EXIT_FORWARD` | 주차 공간에서 직진으로 빠져나오기 |
| `EXIT_TURN` | 원래 주행 방향으로 90도 우회전 |
| `EXIT_STRAIGHT` | OUT 라인까지 직진 |

출차 실패는 감점 대상이므로 주차만큼 중요하게 설계해야 합니다.

### 7-2. 차동 회전 금지

> 반드시 조향으로 주차해야 하며, 차동(좌우 바퀴 속도차)을 이용한 회전 주차는 불허.

**좌후륜과 우후륜의 듀티는 항상 동일해야 합니다.** 방향 전환은 오직 조향 모터로만 수행합니다.

구현상 좌우를 따로 지정할 수 있는 API를 남겨두면 실수 여지가 생기므로, **`set_speed(v)` 단일 함수**로만 구동을 제어하고 좌우 개별 제어 함수는 노출하지 않습니다.

### 7-3. 출발 위치·장애물 배치 추첨 → 하드코딩 금지

> 장애물 차량 위치는 대회 당일 2가지 옵션 중 추첨, 출발 위치는 4곳 중 추첨. **위치 선정 후 프로그램 수정 불가.**

거리나 시간을 하드코딩하면 추첨 결과에 따라 그대로 실패합니다. **모든 상태 전환은 센서 이벤트 기반**이어야 하며, 시간 기반은 다음 두 곳에만 예외적으로 사용합니다.

- 각 상태의 **타임아웃**(안전장치)
- `EXIT_TURN`의 90도 회전 (초음파로는 각도를 측정할 방법이 없음)

### 7-4. 충돌·이탈 방지

충돌은 회당 감점이 크므로, **모든 후진 구간(`REVERSE_RIGHT`, `ALIGN_REVERSE`)에서 S9·S11을 상시 감시**하여 `REAR_STOP_CM` 이내로 접근하면 즉시 전체 정지하고 `ABORT`로 전이합니다.

또한 차량의 모든 바퀴가 회색 영역을 벗어나면 실격이므로, `EXIT_FORWARD`에서 필요 이상으로 멀리 전진하지 않도록 타임아웃을 짧게 잡습니다.

### 7-5. 주요 치수 (임계값 설계 기준)

| 항목 | 치수 |
|---|---|
| 주차 공간 | 950 × 1500 mm |
| 도로 폭 | 850 mm |
| 차선 폭 | 50 mm |
| 제한 시간 | 4분 |

### 7-6. 경기 당일 파일 형식

개발은 노트북으로 하되, 최종 제출은 캘리브레이션·테스트 셀을 제외한 `.py`로 변환합니다.

## 8. 센서 신호 처리 사양

### 8-1. 처리 파이프라인

```
raw tick count (하드웨어 IP 레지스터)
  → 거리(cm) 변환:  ticks / CLOCK_HZ × 34300 / 2
  → timeout이면 400cm로 치환   ("감지 범위 내 반사면 없음")
  → 레인지 클리핑 (2 ~ 400cm, 데이터시트 스펙)
  → 새 측정 판별 (tick 변화 감지)
  → 중앙값 필터 (창 5)
  → ┬→ 밴드 변환 (5단계 + 히스테리시스)  → 상태 전환 판정용
    └→ 연속값 (cm) 그대로               → 정렬 제어(S0-S2 각도)용
```

### 8-2. 5단계 밴드와 히스테리시스

초음파의 절대 거리값은 신뢰도가 낮으므로, **상태 전환 판정에는 거리를 5단계 구간(밴드)으로 양자화**해서 사용합니다.

| 밴드 | 이름 | 기본 경계 |
|---|---|---|
| 0 | 아주 가까움 | ~ 15 cm |
| 1 | 가까움 | 15 ~ 30 cm |
| 2 | 적당함 | 30 ~ 60 cm |
| 3 | 멈 | 60 ~ 120 cm |
| 4 | 아주 멈 | 120 cm ~ |

- **감지 판정**: 밴드 ≤ `DETECT_BAND`
- **해제 판정**: 밴드 ≥ `CLEAR_BAND`

**히스테리시스 필수**: 경계 근처에서 값이 딸랑거리면 상태 전환이 오작동합니다. 올라갈 때는 `경계 + hyst`를 넘어야 하고, 내려올 때는 `경계 − hyst` 아래로 떨어져야 밴드가 바뀝니다.

**센서별 감지 임계 오버라이드**: S9·S11은 45도로 옆 차량을 비스듬히 보기 때문에 같은 상황에서도 측면 센서보다 거리가 근본적으로 길게 나옵니다. 측면 센서와 동일한 임계값을 쓰면 코너 센서가 영영 "감지" 상태로 바뀌지 않으므로, **S9·S11만 별도의 감지 임계 밴드**를 지정해야 합니다.

### 8-3. 갱신 주기와 필터 지연

초음파 IP는 5개 센서를 **라운드로빈으로 순차 측정**합니다.

| 항목 | 값 |
|---|---|
| TRIG 펄스 | 10 µs |
| ECHO 대기 타임아웃 | 30 ms |
| 센서 간 GAP | 30 ms |
| 센서 1개 (정상 감지) | 약 33 ms |
| 센서 1개 (타임아웃) | 약 60 ms |
| **5채널 한 바퀴** | **약 0.16 ~ 0.28 s** |

- 감지되는 센서가 많을수록 한 바퀴가 **빨라집니다** (타임아웃 센서가 60ms를 잡아먹기 때문)
- 따라서 **주기는 주변 환경에 따라 실시간으로 변합니다**
- 중앙값 필터 창 5의 지연 = 2샘플 ≈ **0.34초**
- 15 cm/s 주행 시 약 **5 cm 뒤처진 정보**

**정지 임계값은 이 지연을 보상해서 설정해야 합니다.**

```
실제 정지 임계값 = 목표 거리 + (주행속도 × 필터 지연) + 제동 여유
```

### 8-4. 중복 측정 배제

파이썬 폴링 주기(0.05초)가 하드웨어 측정 주기(약 0.17초)보다 빠르고, `valid` 비트는 다음 차례까지 계속 1로 유지되므로 **같은 측정값을 여러 번 읽게 됩니다.**

그대로 필터 창에 넣으면 창 5칸이 중복으로 채워져 실질적으로 창 2~3 수준의 필터링밖에 되지 않습니다. **직전에 읽은 tick 값과 다를 때만** 새 측정으로 인정해 창에 추가해야 합니다.

## 9. 조향·구동 제어 사양

### 9-1. 조향 폐루프

핸들 축의 가변저항 값을 SPI ADC로 읽어 현재 조향각을 파악하고, 목표값에 도달할 때까지 조향 모터를 **고정 듀티**로 구동하는 단순 폐루프(bang-bang)를 사용합니다. P 제어 등 비례 제어는 쓰지 않습니다.

```
현재 조향값 = ADC를 −7 ~ +7 로 선형 매핑     (+ 가 우조향)
오차 = 목표 − 현재
|오차| ≤ STEER_DEADBAND  →  조향 모터 정지
오차 > 0                  →  우조향 모터 ON
오차 < 0                  →  좌조향 모터 ON
```

- **`ADC_MOST_LEFT` / `ADC_MOST_RIGHT`는 반드시 실측**해야 합니다
- **데드밴드 필수**: 없으면 목표 근처에서 좌우로 계속 헌팅합니다

### 9-2. 조향 목표값의 두 가지 용도

| 용도 | 목표값 | 방식 |
|---|---|---|
| 큰 기동 (좌회전, 후진 진입, 출차 회전) | `0`, `±STEER_FULL`(=±7) | **블로킹** — 바퀴가 다 꺾인 뒤 주행 시작 |
| 정렬 후진 | `0`, `±STEER_ALIGN_STEP`(=±2~4) | **논블로킹** — 매 루프 1스텝씩, 주행하면서 갱신 |

큰 기동에서 블로킹을 쓰는 이유는, 바퀴가 덜 꺾인 채로 움직이면 회전 궤적이 매번 달라져 재현성이 없어지기 때문입니다.

### 9-3. 구동 속도

DC 모터는 듀티가 낮으면 아예 회전하지 않는 구간이 있으므로, **실제로 안정적으로 굴러가는 최소 듀티를 실측**한 뒤 여유를 더해 사용합니다.

| 상수 | 용도 |
|---|---|
| `SPEED_FORWARD` | 1·2단계 탐색 전진 |
| `SPEED_REVERSE` | 4단계 후진 진입 |
| `SPEED_ALIGN` | 5·6·7단계 정렬 후진 (**가장 느리게**) |
| `SPEED_EXIT` | 출차 |

## 10. 최종 상태머신 사양

### 10-1. 전체 흐름

```
INIT              필터 창이 찰 때까지 대기
 ↓
SEARCH_GAP        중립 조향 전진.  S2 감지 → S2·S0 모두 비면 정지
 ↓
SWING_LEFT        최대 좌조향 전진.  S0 재감지 → 다시 비면 정지
 ↓
REVERSE_RIGHT     최대 우조향 후진.  S2 해제 → S2 감지되면 정지
 ↓
ALIGN_REVERSE     S0−S2 각도 정렬 저속 후진.  S9·S11 감지 → 모두 비면 정지
 ↓
HOLD              3~5초 정차 (규정)
 ↓
EXIT_FORWARD      중립 조향 전진.  S0·S8 감지 → 모두 비면 정지
 ↓
EXIT_TURN         최대 우조향 전진 (시간 기반 90도)
 ↓
EXIT_STRAIGHT     중립 조향 직진 (OUT 라인 통과)
 ↓
DONE

※ 어느 상태에서든 타임아웃 또는 후방 근접 발생 시 → ABORT (전체 정지)
```

### 10-2. 상태별 사양표

| 상태 | 조향 | 구동 | 종료 조건 |
|---|---|---|---|
| `SEARCH_GAP` | 0 | 전진 | S2 감지 → **S2·S0 모두 해제** |
| `SWING_LEFT` | −7 | 전진 | S0 감지 → **S0 해제** |
| `REVERSE_RIGHT` | +7 | 후진 | S2 해제 → **S2 감지** |
| `ALIGN_REVERSE` | S0−S2 각도 ±2~4 | 저속 후진 | (최소시간 경과 + S9·S11 감지) → **S9·S11 모두 해제** |
| `HOLD` | 0 | 정지 | 3~5초 경과 |
| `EXIT_FORWARD` | 0 | 전진 | S0·S8 감지 → **S0·S8 모두 해제** |
| `EXIT_TURN` | +7 | 전진 | 설정 시간 경과 |
| `EXIT_STRAIGHT` | 0 | 전진 | 설정 시간 경과 |

### 10-3. 2단계 래치 (핵심 설계)

각 상태의 종료 조건은 단순히 "지금 센서가 어떤가"가 아니라 **"기대한 순서를 한 번 거쳤나"** 입니다.

- **감지 → 해제**: `SEARCH_GAP`, `SWING_LEFT`, `ALIGN_REVERSE`, `EXIT_FORWARD` — 한 번 감지한 뒤에 비어야 종료
- **해제 → 감지**: `REVERSE_RIGHT` — 진입 직후 S2가 이미 근접해 있어도 오종료되지 않도록, 먼저 S2가 비었음을 확인한 뒤 다시 감지되면 종료

**이유**: 중앙값 필터는 상태가 바뀌어도 이전 구간의 값을 창에 들고 있습니다. 현재 상태만 보면 상태에 진입하자마자 직전 구간의 잔상 때문에 조건이 즉시 만족되어 **그 단계를 통째로 건너뜁니다.**

특히 `ALIGN_REVERSE`가 치명적입니다. 진입 시점에 S9·S11이 아직 옆 차량을 보지 못한 상태라면, 래치 없이는 **깊이가 전혀 확보되지 않은 채 주차가 종료**됩니다. 실제 시뮬레이션에서 정렬 후진이 0.57초 만에 끝나는 것을 확인했습니다.

`ALIGN_REVERSE`에는 추가로 **최소 지속 시간**(`ALIGN_MIN_SECONDS`)을 두어 이중으로 방어합니다.

### 10-4. 출차 상세

**`EXIT_FORWARD`에서 S0·S8(둘 다 후방 측면)을 사용하는 이유**

주차된 차량은 코가 도로를 향하고 있으므로, 전진으로 빠져나올 때 **앞쪽 센서가 먼저 자유로워지고 뒷부분이 마지막에 빠져나옵니다.** S2(전측면)가 풀렸다고 회전을 시작하면 아직 두 차 사이에 있는 뒷부분이 옆 차량을 긁습니다. **뒤쪽 센서 두 개(S0, S8)가 모두 풀렸을 때**가 안전한 회전 시점입니다.

**`EXIT_TURN`의 시간 기반 회전**

초음파만으로는 차량 방향(heading)을 측정할 수 없으므로 90도 회전은 시간으로 처리할 수밖에 없습니다. 다만 시간 기반은 **배터리 전압에 취약**하므로:

- 캘리브레이션은 **경기 직전 배터리 상태**에서 수행
- 회전 후 직진 거리를 넉넉히 확보 (OUT 라인은 앞바퀴만 넘으면 성공이므로 정밀도 요구가 낮음)
- 가능하면 회전 후 S0·S2에 우측 장애물 차량이 다시 잡히는지로 방향을 개략 검증

## 11. 튜닝 상수 일람

모든 튜닝 값은 코드 상단 한 곳(`CONFIG`)에 모으고, 나머지 코드는 이를 참조만 하도록 구성합니다. 현장에서 고칠 곳이 한 군데여야 디버깅 속도가 확보됩니다.

| 분류 | 상수 | 결정 방법 |
|---|---|---|
| 센서 매핑 | `SENSOR_CH` | 실제 배선 확인 |
| 밴드 | `BAND_EDGES_CM` | 장애물 감지 시 / 빈 공간 시 거리 실측 후 그 사이를 가르도록 |
| 밴드 | `BAND_HYST_CM` | 값이 딸랑거리면 키움 |
| 밴드 | `DETECT_BAND_OVERRIDE` | S9·S11의 45도 각도 보정 |
| 조향 | `ADC_MOST_LEFT` / `ADC_MOST_RIGHT` | 핸들을 좌우 끝까지 돌려 ADC 실측 |
| 조향 | `STEER_DUTY` | 조향이 느리면 키움 |
| 조향 | `STEER_DEADBAND` | 헌팅하면 키움 |
| 조향 | `STEER_ALIGN_STEP` | 2에서 시작, 공간 내에 각도가 안 펴지면 3~4로 |
| 구동 | `SPEED_*` | 최소 구동 듀티 실측 + 여유 |
| 정렬 | `ALIGN_DEADBAND_CM` | 각도가 좌우 헌팅하면 키움 |
| 정렬 | **`ALIGN_SIGN`** | **오차가 줄지 않고 커지면 −1로 뒤집기** |
| 정렬 | `ALIGN_MIN_SECONDS` | 정렬이 너무 일찍 끝나면 키움 |
| 안전 | `REAR_STOP_CM` | 후진이 너무 깊게 들어가면 키움 |
| 출차 | `EXIT_TURN_SECONDS` | 회전각이 부족/과하면 조정 |
| 출차 | `EXIT_STRAIGHT_SECONDS` | OUT 라인에 못 미치면 늘림 |

### 현장 검증 순서

1. 모터 6개가 이름대로 도는지 개별 확인
2. 조향 ADC 좌우 끝값 실측
3. 초음파 5채널 생존 확인
4. 최소 구동 듀티 실측
